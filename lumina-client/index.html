Introduction

LuminaClient is a sophisticated ghost client for Minecraft 1.19.4, built on the Fabric modding framework. The name stands for "Light Utilization Management Integrating Notable Acceleration" - a fitting description for a client that seamlessly integrates powerful features while maintaining a low profile.

As an alpha-stage project, LuminaClient showcases advanced software architecture patterns and deep integration with Minecraft's client-side codebase through the innovative use of mixins, event systems, and modular design.

Event Usage Example
public class ExampleModule extends Module {
    @EventTarget(priority = 2)
    public void onUpdate(EventUpdate event) {
        // Execute logic every game tick (20 times/second)
    }

    @EventTarget(priority = 1)
    public void onPacket(PacketSendEvent event) {
        if (shouldCancelPacket()) {
            event.cancel(); // Prevents packet from being sent
        }
    }
}

Mixin System: The Foundation

The mixin system is what makes LuminaClient possible. Mixins allow the client to inject custom code into Minecraft's classes at runtime without modifying the original JAR files through bytecode manipulation.

ClientPlayNetworkHandlerMixin

Packet Interception

Intercepts packet sending and chat messages, enabling packet logging, modification, and cancellation

MinecraftClientMixin

Game Loop Integration

Main game loop integration for tick updates and global state management

InGameHudMixin

HUD Rendering

GameRendererMixin

Camera & View

Camera manipulation for NoHurtCam, Freecam, and FOV modifications

WorldRendererMixin

World Rendering

Selective block rendering for XRay, ESP, and custom world overlays

KeyboardMixin

Input Capture

Keyboard input capture for module keybind system and GUI activation

Mixin Injection Example
@Inject(method = "sendPacket", at = @At("HEAD"), cancellable = true)
private void onSendPacket(Packet<?> packet, CallbackInfo ci) {
    PacketSendEvent event = new PacketSendEvent(packet);
    event.call();
    if (event.isCancelled()) {
        ci.cancel(); // Prevents packet from being sent
    }
}

Notable Technical Implementations

Rotation Calculation

Combat modules use advanced trigonometry for realistic player rotation. The system calculates yaw (horizontal) and pitch (vertical) angles using atan2 for quadrant-aware positioning. 

// Calculate yaw (horizontal rotation)
float yaw = (float)(Math.toDegrees(Math.atan2(deltaZ, deltaX)) - 90.0);

// Calculate pitch (vertical rotation)
float pitch = (float)(-Math.toDegrees(Math.atan(deltaY / distance)));

Animation System

Custom DecelerateAnimation class provides smooth UI transitions for module enable/disable, panel expansion, and visual effects. Animations use time-based interpolation for consistent frame-rate-independent motion.

Color Interpolation

The Arraylist module uses HSB color cycling with time-based gradients to create smooth rainbow effects. Colors interpolate between two hues with configurable speed and spread for each module. 

// Time-based color cycling
float hue = (System.currentTimeMillis() / colorSpeed) % 1.0f;
float offset = index * colorSpread;
Color rainbow = Color.getHSBColor((hue + offset) % 1.0f, 1.0f, 1.0f);

Fake Player Rendering

The Blink module renders a client-side fake player entity to show where the real player appears to others. The fake player copies skin, position, rotation, and inventory for realistic visualization.

Block Data Caching

Scaffold pre-calculates optimal block placement positions by checking all 6 faces for valid placement. This caching system reduces per-tick calculations and improves performance.

Configuration & Persistence

LuminaClient uses a JSON-based configuration system for persisting module states and settings. Configuration is loaded asynchronously on startup and saved automatically on changes.

Location: %appdata%\.minecraft\luminaConfig.json
Format: JSON with module names as keys
Auto-Save: On module toggle, setting change, shutdown, or HUD reposition
